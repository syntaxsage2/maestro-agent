# 前端用户认证功能说明

## 功能概述

前端现在已经集成了完整的用户认证系统，包括：

✅ **用户登录** - 使用用户名和密码登录
✅ **用户注册** - 创建新账号
✅ **自动认证** - 刷新页面后自动恢复登录状态
✅ **JWT Token 管理** - 自动在 API 请求中添加认证令牌
✅ **用户菜单** - 显示用户信息、切换主题、退出登录
✅ **主题切换** - 支持亮色/暗色主题切换
✅ **会话管理** - 退出时自动清空聊天记录

## 使用流程

### 1. 用户首次访问

当用户首次访问应用时，会看到**登录页面**：
- 输入用户名和密码
- 点击"登录"按钮
- 或点击"立即注册"切换到注册页面

### 2. 用户注册

在注册页面，用户需要填写：
- **用户名**（必填，至少 3 个字符）
- **邮箱**（必填，需要有效的邮箱格式）
- **姓名**（可选）
- **密码**（必填，至少 6 个字符）
- **确认密码**（必填，需要与密码一致）

注册成功后会自动登录并跳转到聊天界面。

### 3. 已登录状态

登录成功后：
- 自动保存认证信息到 `localStorage`
- 刷新页面会自动恢复登录状态
- 所有 API 请求会自动携带 JWT Token
- 右上角显示用户头像和用户名

### 4. 用户菜单功能

点击右上角的用户头像，会弹出菜单：

#### 📋 个人信息
- 显示用户姓名和邮箱
- 可以点击查看详细信息（待实现）

#### 🌓 主题切换
- 切换到亮色：适合白天使用
- 切换到暗色：适合夜间使用
- 主题设置会自动保存

#### 🚪 退出登录
- 清除本地认证信息
- 清空所有聊天记录
- 返回登录页面

## Token 管理

### 自动添加 Token

API 客户端（`src/api/client.ts`）已配置请求拦截器：

```typescript
// 请求拦截器 - 自动添加 JWT token
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

### Token 过期处理

响应拦截器会自动处理 Token 过期（401 错误）：
- 清除本地存储的认证信息
- 自动跳转到登录页面
- 显示"登录已过期，请重新登录"提示

## 安全特性

### 1. 密码验证
- 最少 6 个字符
- 注册时需要确认密码

### 2. Token 存储
- 使用 `localStorage` 存储 Token
- 刷新页面后自动恢复

### 3. 自动登出
- Token 过期时自动登出

- 每个用户的聊天记录独立存储
- 退出登录时清空当前用户的所有数据

## 开发建议

### 后端 API 配置

确保后端已启动并配置正确的 CORS：

```python
# FastAPI CORS 配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],  # Vite 默认端口
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 环境变量

在 `.env` 文件中配置 API 地址：

```env
VITE_API_BASE_URL=http://localhost:8000
VITE_API_TIMEOUT=30000
```

## 常见问题

### Q: 登录后刷新页面会掉线吗？
A: 不会。Token 和用户信息都保存在 `localStorage` 中，刷新页面会自动恢复登录状态。

### Q: Token 有效期是多久？
A: 根据后端配置，默认是 7 天（参考 AUTH_GUIDE.md）。

### Q: 如何切换主题？
A: 点击右上角用户头像，在菜单中点击"切换到亮色/暗色"。

### Q: 退出登录后数据还在吗？
A: 退出登录会清空当前用户的聊天记录，但用户信息和长期记忆保存在后端数据库中，重新登录后可以恢复。

### Q: 支持多用户吗？
A: 是的，每个用户有独立的账号和数据，完全隔离。
