## ? 核心目录详解

### `/src/api` - API 调用层
负责所有与后端的通信。

**client.ts**
- axios 实例配置
- 请求/响应拦截器
- 错误处理
- API 端点常量

**chat.ts**
- 发送普通消息
- 发送多模态消息
- 获取流式聊天 URL

**history.ts**
- 获取会话历史
- 获取工具调用历史
- 删除会话

**hitl.ts**
- 获取中断状态
- 批准/拒绝操作
- 恢复执行

**system.ts**
- 健康检查
- 获取工具列表

### `/src/components` - UI 组件层
所有可复用的 React 组件。

#### `/Chat` - 聊天相关
- **ChatContainer** - 聊天主容器，整合消息列表和输入框
- **MessageList** - 消息滚动列表，包含欢迎界面
- **MessageItem** - 单条消息渲染，支持 Markdown
- **MessageInput** - 消息输入，支持多行和快捷键
- **NodeStatus** - 显示当前执行的节点
- **ImageUpload** - 图片拖拽上传组件

#### `/Sidebar` - 侧边栏
- **Sidebar** - 会话列表和功能快捷入口

#### `/HITL` - 人机协同
- **ConfirmDialog** - 高风险操作确认对话框

#### `/Layout` - 布局
- **Layout** - 应用整体布局
- **Header** - 顶部导航栏

### `/src/hooks` - 自定义 Hooks
封装复用的业务逻辑。

**useChat.ts**
- 发送文本消息
- 发送图片消息
- 处理流式响应
- 管理加载状态

**useSSE.ts**
- 建立 SSE 连接
- 处理事件流
- 错误处理

**useHITL.ts**
- 轮询中断状态
- 批准/拒绝操作
- 管理确认流程

### `/src/store` - 状态管理
使用 Zustand 管理全局状态。

**chatStore.ts**
- 会话列表
- 当前会话
- 流式响应状态
- 节点执行状态
- 侧边栏显示状态
- 增删改查操作
- 持久化到 localStorage

### `/src/types` - 类型定义
所有 TypeScript 类型和接口。

**api.ts**
- 请求/响应类型
- SSE 事件类型
- 枚举类型
- 前端专用类型

### `/src/utils` - 工具函数
通用的辅助函数。

**time.ts**
- 相对时间格式化（刚刚、5分钟前）
- 完整时间格式化
- 简短时间格式化

**storage.ts**
- localStorage 操作封装
- 会话存储
- 主题设置
- 用户配置

## ? 数据流向

### 1. 用户发送消息流程
```
用户输入
  ↓
MessageInput 组件
  ↓
useChat Hook
  ↓
useSSE Hook
  ↓
API (chat.ts)
  ↓
后端服务器
  ↓
SSE 事件流
  ↓
chatStore 更新
  ↓
MessageList 刷新
```

### 2. 会话管理流程
```
用户操作
  ↓
Sidebar 组件
  ↓
chatStore actions
  ↓
localStorage (storage.ts)
  ↓
持久化存储
```

### 3. HITL 确认流程
```
后端中断
  ↓
useHITL 轮询检测
  ↓
ConfirmDialog 弹出
  ↓
用户确认/拒绝
  ↓
API (hitl.ts)
  ↓
后端继续执行
```

## ? 代码规范

### 命名约定
- 组件：PascalCase (MessageItem.tsx)
- 函数/变量：camelCase (sendMessage)
- 常量：UPPER_SNAKE_CASE (API_BASE_URL)
- 类型：PascalCase (ChatRequest)

### 文件组织
- 每个组件一个文件
- 相关组件放在同一目录
- 导出使用具名导出

### TypeScript
- 所有函数标注参数和返回类型
- 避免使用 any
- 使用接口定义数据结构
